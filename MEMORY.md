## 用户反馈的关键问题（必须解决）

### PROBLEM-2026-03-01-01: 卡死不动 💤
type: problem
area: execution

现象：
- 执行脚本后长时间无输出
- 使用后台执行（&）导致无法看到错误
- 反复sleep检查不存在的文件
- 浪费时间（本次30-40分钟）

原因分析：
1. 后台执行无反馈 - 用`&`后台运行，看不到输出和错误
2. 网络操作超时 - git/npm下载等待时间长
3. 死循环/阻塞 - 脚本卡在某个步骤

解决方案：
```bash
# ❌ 错误做法
node script.js &

# ✅ 正确做法 - 前台运行带超时
timeout 120 node script.js || echo "命令超时或失败"

# ✅ 正确做法 - 检查进程
node script.js &
PID=$!
sleep 10
ps -p $PID > /dev/null || echo "进程已退出"
```

执行原则更新：
1. **不要后台执行** - 除非确实需要，使用前台运行
2. **添加超时限制** - 所有命令使用timeout
3. **实时汇报进展** - 每个步骤立即输出状态
4. **定期检查状态** - 每10-30秒检查一次
5. **失败立即止损** - 超时立即停止，切换方案
